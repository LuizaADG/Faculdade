	No quicksort paralelo, a variação entre cada uma das threads é muito pequena para o mesmo tamanho. Isso se deve ao pequeno tamanho de cada um dos arrays para a tarefa. Apesar de não ser muito significativo, nota-se uma diferença na velocidade para a mesma thread com valores de tamanho de array diferentes, sendo que quanto maior o array, mais devagar a sua ordenação fica. Também se nota que a variação de valor se altera dependendo de como a linha recursiva é alterada. Caso se divida por2 a velocidade é a mais rápida, e se subraírmos 1, a velocidade é a mais devagar.
